OSRSFlipper 1.0.9 — Database Compaction and Maintenance History

Release type: database maintenance and safety release

Included:
- Added Database Compaction to Admin > Data Health.
- Added Preview Compact Database action.
- Added SQLite page/free-page compaction preview.
- Added estimated reclaimable database space.
- Added estimated compacted database size.
- Added fresh backup status to compaction preview.
- Added Maintenance History to Admin > Data Health.
- Added data_maintenance_events table.
- Added maintenance event logging for compaction previews and compacted-copy creation.
- Added guarded compacted database copy creation.
- Added exact compaction confirmation requirement: COMPACT DATABASE.
- Added fresh safety-backup requirement before compacted-copy creation.
- Added VACUUM INTO compacted-copy workflow.
- Added PRAGMA integrity_check verification for compacted copies.
- Added compacted database copy metadata files.
- Added .gitignore entries for local compacted database copies.
- Kept active database replacement manual-only.

Not included:
- Automatic GE trading.
- Automatic buy/sell/cancel actions.
- Automatic repricing.
- Automatic database replacement or live database swap.
- Automatic unattended compaction.
- Automatic VACUUM against the active database.
- Machine-learning prediction model.
- Private database files.
- Local database backups.
- Local compacted database copies.
- Saved logs.
- Exports.
- Runtime cache files.
- .env.
- API keys.

Notes:
- This release completes the safe database maintenance workflow started in 1.0.8.
- Compaction preview estimates reclaimable SQLite space without changing the database.
- Guarded compacted-copy creation uses VACUUM INTO to create a separate compacted copy.
- The active database is not replaced while the dashboard is running.
- Compacted copies are verified with PRAGMA integrity_check.
- Manual replacement/swap should only be done while the dashboard is fully stopped.
- Maintenance History provides an audit trail for previews and compacted-copy actions.

OSRSFlipper 1.0.8 — Data Automation and Retention Safety

Release type: data maintenance and safety release

Included:
- Added Metrics Automation to Admin > Data Health.
- Added daily_item_metrics freshness checks.
- Added Refresh Stale Metrics action.
- Added automatic stale-metrics refresh before loading Item Trends.
- Added Retention Safety to Admin > Data Health.
- Added raw scan_results retention preview.
- Added Keep forever, 120-day, 90-day, 60-day, and 30-day retention preview options.
- Added cleanup preview table showing cutoff date, rows affected, rows retained, and estimated impacted size.
- Added Database Backup to Admin > Data Health.
- Added Create Safety Backup action using the SQLite backup API.
- Added database backup list and verification table.
- Added .gitignore entries for local database safety backups.
- Added guarded raw scan_results cleanup.
- Added exact cleanup confirmation requirement: DELETE OLD SCANS.
- Added fresh-backup requirement before cleanup can run.
- Added daily_item_metrics rebuild before old raw scan rows are deleted.
- Preserved aggregate trend history before cleanup.
- Kept database compaction/VACUUM out of this release.

Not included:
- Automatic GE trading.
- Automatic buy/sell/cancel actions.
- Automatic repricing.
- Automatic unattended cleanup.
- Automatic VACUUM or database compaction.
- Machine-learning prediction model.
- Private database files.
- Local database backups.
- Saved logs.
- Exports.
- Runtime cache files.
- .env.
- API keys.

Notes:
- Cleanup is intentionally guarded and cannot run unless a fresh safety backup exists.
- Cleanup is also blocked unless the exact confirmation text is entered.
- Keep forever blocks cleanup.
- The cleanup action only targets old raw scan_results rows based on the selected retention window.
- daily_item_metrics is rebuilt before cleanup so long-term aggregate trend history remains available.
- SQLite file size may not shrink immediately after cleanup because VACUUM/compaction is not performed in this release.
- This release focuses on keeping the data pipeline current and making raw scanner growth safer to manage.
